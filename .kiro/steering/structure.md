# Project Structure

## Root Directory Layout

```
├── ez                          # Main CLI executable (generated by Bashly)
├── src/                        # Source files for CLI generation
├── docker/                     # Shared Docker configurations
├── template/                   # Docker templates for Laravel apps
├── apps/                       # Laravel application instances
├── shared/                     # Shared services data
├── test/                       # Comprehensive testing suite
├── temp/                       # Temporary environment files
└── tmp/                        # Runtime temporary files
```

## Key Directories

### `/src/` - CLI Source Code
- `bashly.yml`: CLI command definitions and structure
- `*_command.sh`: Individual command implementations
- `lib/`: Reusable shell functions and utilities
- `before.sh`: Global setup and initialization

### `/docker/` - Shared Infrastructure
- `compose-shared.yml`: Shared services (Nginx PM, MySQL, phpMyAdmin)
- `common-shared.yml`: Common service definitions
- `mysql.cnf`: MySQL configuration
- `docker.env`: Environment variables for shared services

### `/template/` - Laravel App Templates
- `laravel.Dockerfile`: Production Laravel container
- `laravel-dev.Dockerfile`: Development container with live reload
- `laravel-test.Dockerfile`: Testing environment container
- `compose-laravel.yml`: Laravel app service definitions
- `nginx/`: Nginx configuration files
- `*.ini`: PHP configuration files
- `entrypoint*.sh`: Container startup scripts

### `/apps/` - Application Instances
Each Laravel app gets its own directory:
```
apps/{app_name}/
├── env/
│   ├── app.env              # App-specific variables
│   ├── laravel.env          # Laravel framework defaults
│   ├── {environment}.env    # Environment overrides
│   └── generated/           # Merged environment files
├── src-{env}/               # Source code per environment
└── storage-{env}/           # Persistent storage volumes
```

### `/test/` - Testing Framework
- `unit/`: Function-level tests
- `integration/`: Full workflow tests
- `security/`: Security-focused tests
- `fixtures/`: Test data and mock files
- `approvals/`: Approved test outputs
- `tmp/`: Test temporary files

## File Naming Conventions

### Scripts
- Command implementations: `{service}_{action}_command.sh`
- Library functions: `{function_name}.sh` in `src/lib/`
- Test files: `test_{category}_{subject}.sh`

### Docker Files
- Dockerfiles: `{service}-{environment}.Dockerfile`
- Compose files: `compose-{service}.yml`
- Common definitions: `common-{service}.yml`

### Environment Files
- App configuration: `app.env`
- Environment overrides: `{environment}.env` (dev.env, test.env, etc.)
- Generated merged: `generated/{environment}.env`

## Configuration Hierarchy

Environment variables are merged in this order (later overrides earlier):
1. `laravel.env` - Laravel framework defaults
2. `docker.env` - Docker infrastructure settings
3. `app.env` - Application-specific settings
4. `{environment}.env` - Environment-specific overrides

## Development Workflow

1. **CLI Development**: Modify `src/bashly.yml` and command files, then regenerate with `bashly generate`
2. **Docker Changes**: Update templates in `/template/`, changes apply to new deployments
3. **App Deployment**: Apps are created in `/apps/` with environment-specific configurations
4. **Testing**: All changes should include tests in appropriate `/test/` subdirectories

## Important Notes

- All scripts require sudo privileges for Docker operations
- Environment-specific source code is stored in `src-{environment}` directories
- Shared services run on a common Docker network
- Port assignments are automatic to prevent conflicts
- Git branches map to environments (main→production, environment→environment)
