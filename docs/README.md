## Documentation Standards

All documentation in this project follows these principles:
- **Concise**: Main README is brief with links to detailed docs
- **Accurate**: Updated with each major change
- **Consistent**: Same terminology and formatting throughout
- **Accessible**: Clear language, good examples, troubleshooting included

## Contributing to Documentation

When updating documentation:
1. Keep README.md concise (~250 lines)
2. Put detailed information in topic-specific docs
3. Update this index when adding new docs
4. Ensure all links work
5. Update "Last Updated" dates
6. Test all code examples
